## 1. Python Basics

### Definition
Python is a high-level, interpreted, general-purpose programming language known for readable syntax and strong standard library support. It is commonly used in scripting, web development, automation, data analysis, and machine learning.

### Why Python Is Popular in Interviews
- Syntax is clean and close to plain English.
- It supports multiple programming styles: procedural, object-oriented, and functional.
- It allows rapid problem solving in coding rounds.
- It has built-in data structures that reduce code size.
- It is heavily used in industry for backend, automation, testing, analytics, and AI work.

### Basic Features
- Dynamically typed: variable types are decided at runtime.
- Interpreted: code is executed line by line by the Python interpreter.
- Case-sensitive: `name` and `Name` are different identifiers.
- Indentation-based: blocks are defined using indentation, not braces.
- Large standard library: Python documentation highlights the tutorial, language reference, and standard library as core learning resources [2].

### Example
```python
name = "Sarwesh"
age = 20
print(name)
print(age)
```

### Important Terms
- Variable: a name that refers to a value.
- Identifier: a valid name for variables, functions, classes, and more.
- Keyword: reserved words like `if`, `else`, `for`, `class`, and `def` that cannot be used as identifiers [3].
- Expression: a combination of values, operators, or function calls that produces a result [3].
- Statement: an instruction executed by Python.

### Interview Questions and Answers
**Q1. Is Python compiled or interpreted?**  
Python is commonly described as an interpreted language because code is executed by the interpreter, although internally it may first convert source code to bytecode.

**Q2. What is the role of indentation in Python?**  
Indentation defines code blocks. Missing or inconsistent indentation can produce errors such as `IndentationError` [4][5].

**Q3. What are Python keywords?**  
Keywords are reserved words with special meaning in the language, such as `if`, `for`, `return`, and `class` [3].

**Q4. Why is Python preferred in coding interviews?**  
Because it has concise syntax, powerful built-in data structures, and quick implementation for logic-heavy questions.

## 2. Data Types

### Definition
A data type classifies the kind of value a variable stores and determines what operations are valid on that value [3].

### Common Built-in Data Types
- `int`: whole numbers, example `10`
- `float`: decimal numbers, example `3.14`
- `complex`: complex numbers, example `2 + 3j`
- `str`: text, example `"hello"`
- `bool`: `True` or `False`
- `list`: ordered, mutable collection
- `tuple`: ordered, immutable collection
- `set`: unordered collection of unique elements
- `dict`: key-value mapping [3]
- `NoneType`: represents no value using `None`

### Mutable vs Immutable
- Mutable objects can be changed after creation, such as lists, sets, and dictionaries.
- Immutable objects cannot be changed after creation, such as integers, strings, and tuples.

### Type Checking
```python
x = 10
print(type(x))
```

### Type Conversion
```python
num = "25"
converted = int(num)
print(converted + 5)
```

### Interview Questions and Answers
**Q1. What is the difference between mutable and immutable types?**  
Mutable objects can be modified after creation, while immutable objects cannot.

**Q2. What is `None` in Python?**  
`None` is a special value representing the absence of a value or a null state.

**Q3. Can Python variables change type?**  
Yes. Python is dynamically typed, so a variable can refer to different types at different times.

**Q4. What happens if incompatible types are used together?**  
Python raises a runtime exception such as `TypeError` [5].

## 3. Lists, Tuples, Sets, Dictionaries

### Lists
A list is an ordered, mutable collection that can store elements of different types.

```python
fruits = ["apple", "banana", "mango"]
fruits.append("orange")
print(fruits)
```

#### Common List Operations
- `append()` adds one item.
- `extend()` adds multiple items.
- `insert()` adds at a specific index.
- `remove()` removes by value.
- `pop()` removes by index and returns the item.
- `sort()` sorts the list.
- `reverse()` reverses the list.

### Tuples
A tuple is an ordered, immutable collection. The Python glossary describes a tuple as a sequence like a list whose contents cannot be changed [3].

```python
dimensions = (1920, 1080)
print(dimensions[0])
```

### Sets
A set is an unordered collection of unique elements.

```python
numbers = {1, 2, 3, 3, 4}
print(numbers)
```

#### Set Use Cases
- Removing duplicates from a collection.
- Fast membership testing.
- Mathematical operations like union and intersection.

### Dictionaries
A dictionary stores key-value pairs and is used for fast lookup by key [3].

```python
student = {
    "name": "Anu",
    "age": 21,
    "branch": "ECE"
}
print(student["name"])
```

### Comparison Table
| Structure | Ordered | Mutable | Duplicates | Access Style |
|---|---|---|---|---|
| List | Yes | Yes | Allowed | Index |
| Tuple | Yes | No | Allowed | Index |
| Set | No | Yes | Not allowed | No indexing |
| Dictionary | Insertion order preserved in modern Python | Yes | Keys unique | Key lookup |

### Interview Questions and Answers
**Q1. Difference between list and tuple?**  
A list is mutable, while a tuple is immutable.

**Q2. Why use a set?**  
Use a set when uniqueness matters or when fast membership checks are needed.

**Q3. Can dictionary keys be mutable?**  
No. Keys must be hashable, so mutable types like lists cannot be used as keys.

**Q4. How do you remove duplicates from a list?**  
Convert the list to a set, then back to a list if needed: `list(set(my_list))`.

**Q5. What is the average time complexity of dictionary lookup?**  
Typically close to O(1) average case because dictionaries are hash-table based.

## 4. Functions

### Definition
A function is a reusable block of code that performs a specific task. Functions improve modularity, readability, and reuse.

### Syntax
```python
def greet(name):
    return f"Hello, {name}"
```

### Types of Function Arguments
- Positional arguments
- Keyword arguments
- Default arguments
- Variable-length arguments: `*args`
- Keyword variable-length arguments: `**kwargs`

### Example
```python
def add(a, b=0):
    return a + b

print(add(5, 3))
print(add(5))
```

### Scope
- Local variable: defined inside a function.
- Global variable: defined outside all functions.
- Python also supports `nonlocal` in nested functions.

### Interview Questions and Answers
**Q1. What is the difference between parameters and arguments?**  
Parameters are variables in the function definition. Arguments are actual values passed during the call.

**Q2. What is `*args`?**  
It allows a function to accept a variable number of positional arguments.

**Q3. What is `**kwargs`?**  
It allows a function to accept a variable number of keyword arguments.

**Q4. What is recursion?**  
Recursion is when a function calls itself to solve a smaller version of a problem.

### Recursion Example
```python
def factorial(n):
    if n == 0 or n == 1:
        return 1
    return n * factorial(n - 1)
```

## 5. OOP Concepts

### Definition
Object-oriented programming organizes code using classes and objects. It helps model real-world entities and supports reusable, maintainable code.

### Core Concepts
- Class: blueprint for creating objects.
- Object: instance of a class.
- Attribute: data stored in an object.
- Method: function defined inside a class.

### Four Pillars of OOP
- Encapsulation: bundling data and methods together.
- Abstraction: hiding implementation details and showing essential behavior.
- Inheritance: creating a new class from an existing class.
- Polymorphism: one interface, multiple behaviors.

### Example
```python
class Animal:
    def speak(self):
        return "Some sound"

class Dog(Animal):
    def speak(self):
        return "Bark"

d = Dog()
print(d.speak())
```

### Special Methods
- `__init__`: constructor-like initializer.
- `__str__`: string representation for users.
- `__repr__`: developer-friendly representation.

### Interview Questions and Answers
**Q1. What is the difference between a class and an object?**  
A class is a template, while an object is an instance created from that template.

**Q2. What is inheritance?**  
Inheritance allows one class to reuse and extend properties and methods of another class.

**Q3. What is method overriding?**  
Method overriding happens when a child class provides its own implementation of a method already defined in the parent class.

**Q4. What is the use of `self`?**  
`self` refers to the current instance of the class and is used to access instance variables and methods.

## 6. Exception Handling

### Definition
Python distinguishes syntax errors from exceptions. Syntax errors are detected while parsing code, while exceptions are errors detected during execution [5].

### Common Exceptions
- `ZeroDivisionError` when dividing by zero [5]
- `NameError` when a variable is not defined [5]
- `TypeError` when an operation uses an inappropriate type [5]
- `ValueError` when a function gets a value of correct type but invalid content
- `IndexError` when sequence index is out of range [4]
- `KeyError` when a dictionary key does not exist
- `FileNotFoundError` when a file cannot be found

### Basic Handling Syntax
```python
try:
    x = int(input("Enter number: "))
    print(10 / x)
except ZeroDivisionError:
    print("Cannot divide by zero")
except ValueError:
    print("Invalid input")
```

### Flow of `try-except`
The Python tutorial explains that the `try` clause executes first, the `except` clause runs only when a matching exception occurs, and unmatched exceptions propagate outward as unhandled exceptions [5].

### `else` and `finally`
- `else` runs if no exception occurs in the `try` block [5].
- `finally` runs whether or not an exception occurs and is commonly used for cleanup [5][6].

### Example with `finally`
```python
try:
    file = open("data.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("File not found")
finally:
    print("Execution completed")
```

### Raising Exceptions
Python allows programmers to force an exception using the `raise` statement [5][6].

```python
age = -1
if age < 0:
    raise ValueError("Age cannot be negative")
```

### Interview Questions and Answers
**Q1. What is the difference between syntax error and exception?**  
Syntax errors are detected before execution, while exceptions occur during execution [5].

**Q2. Why should specific exceptions be caught instead of a broad `except`?**  
The Python tutorial recommends handling exceptions as specifically as possible so unexpected errors can propagate and be handled properly [5].

**Q3. What is the purpose of `finally`?**  
It is used for cleanup actions that must happen regardless of success or failure, such as closing files or network connections [5].

**Q4. Can you re-raise an exception?**  
Yes. Python supports re-raising an exception using `raise` inside an exception handler [5].

## 7. File Handling

### Definition
File handling means creating, opening, reading, writing, and closing files from a Python program.

### Common Modes
- `r`: read
- `w`: write, overwrites file
- `a`: append
- `x`: create new file
- `rb`: read binary
- `wb`: write binary

### Reading a File
```python
file = open("sample.txt", "r")
content = file.read()
file.close()
```

### Writing a File
```python
file = open("sample.txt", "w")
file.write("Hello Python")
file.close()
```

### Best Practice: `with`
The Python tutorial explains that `with` ensures resources like files are cleaned up promptly and correctly, and files are closed after the block finishes even if problems occur during processing [5].

```python
with open("sample.txt", "r") as file:
    content = file.read()
    print(content)
```

### Interview Questions and Answers
**Q1. Why is `with open()` preferred over `open()` and `close()` manually?**  
Because it automatically handles cleanup and closes the file even if an exception occurs [5].

**Q2. Difference between `read()`, `readline()`, and `readlines()`?**  
`read()` gets the whole file, `readline()` gets one line, and `readlines()` gets all lines as a list.

**Q3. What happens in `w` mode if the file already exists?**  
The file is overwritten.

## 8. Modules and Packages

### Definition
A module is a Python file containing code such as functions, variables, or classes. A package is a collection of related modules, usually organized in folders.

### Why They Matter
- Improve code organization.
- Support reuse across projects.
- Allow separation of concerns.
- Help teams maintain large codebases.

### Import Examples
```python
import math
print(math.sqrt(16))

from math import pi
print(pi)
```

### Custom Module Example
Suppose `calc.py` contains:
```python
def square(x):
    return x * x
```
Then another file can use:
```python
import calc
print(calc.square(5))
```

### Interview Questions and Answers
**Q1. What is the difference between a module and a package?**  
A module is a single Python file, while a package is a directory containing multiple related modules.

**Q2. What is the use of `__name__ == "__main__"`?**  
It checks whether the file is being run directly or imported as a module.

**Q3. Why use aliases in imports?**  
Aliases such as `import numpy as np` make code shorter and often more readable.

## 9. Generators and Iterators

### Iterator Definition
An iterator is an object that can be traversed one element at a time and implements the iteration protocol, mainly `__iter__()` and `__next__()`.

### Generator Definition
A generator is a simple way to create iterators using functions with the `yield` keyword. It produces values lazily, meaning one value at a time.

### Iterator Example
```python
nums = [1, 2, 3]
it = iter(nums)
print(next(it))
print(next(it))
```

### Generator Example
```python
def count_up_to(n):
    count = 1
    while count <= n:
        yield count
        count += 1

for value in count_up_to(3):
    print(value)
```

### Benefits of Generators
- Memory efficient for large datasets.
- Values are generated on demand.
- Useful in streams, pipelines, and big-data style processing.

### Interview Questions and Answers
**Q1. Difference between iterator and generator?**  
An iterator is any object following the iteration protocol. A generator is a convenient way to create such an iterator using `yield`.

**Q2. Why are generators memory efficient?**  
Because they do not store all values at once; they generate values only when requested.

**Q3. What happens when a generator finishes?**  
It raises `StopIteration` internally.

## 10. Decorators

### Definition
A decorator is a function that takes another function, adds extra behavior, and returns a new function without changing the original function’s core code.

### Why Decorators Matter
- Reuse common logic like logging, timing, access control, and validation.
- Keep business logic separate from cross-cutting concerns.
- Frequently asked in intermediate Python interviews.

### Example
```python
def decorator_func(original_func):
    def wrapper():
        print("Before function call")
        original_func()
        print("After function call")
    return wrapper

@decorator_func
def display():
    print("Hello")

display()
```

### Interview Questions and Answers
**Q1. What is the purpose of a decorator?**  
It extends or modifies function behavior without rewriting the original function.

**Q2. What does `@decorator_name` mean?**  
It is syntactic sugar for passing a function to a decorator and reassigning the returned function.

**Q3. Where are decorators used in practice?**  
Common uses include logging, authentication, caching, retries, and performance measurement.

## 11. List Comprehensions

### Definition
A list comprehension is a compact syntax for creating a new list from an iterable, optionally with a condition.

### Basic Syntax
```python
squares = [x * x for x in range(5)]
print(squares)
```

### Conditional Example
```python
evens = [x for x in range(10) if x % 2 == 0]
print(evens)
```

### Why Interviewers Ask This
- Tests understanding of iteration and filtering.
- Shows whether the candidate can write clean Pythonic code.
- Helps compare readability versus traditional loops.

### Interview Questions and Answers
**Q1. What is the advantage of list comprehensions?**  
They provide concise and readable syntax for generating lists.

**Q2. Are list comprehensions always better than loops?**  
No. They are good for simple transformations, but complex logic may be clearer with regular loops.

**Q3. Can nested loops be used inside a list comprehension?**  
Yes, but readability can suffer if the logic becomes too complex.

## 12. Lambda Functions

### Definition
A lambda function is a small anonymous function written in a single expression.

### Syntax
```python
add = lambda a, b: a + b
print(add(2, 3))
```

### Common Use Cases
- Short functions passed to `map()`, `filter()`, or `sorted()`.
- Key functions in sorting.
- Small inline transformations.

### Example with `sorted`
```python
students = [("A", 80), ("B", 65), ("C", 90)]
sorted_students = sorted(students, key=lambda x: x[1])
print(sorted_students)
```

### Interview Questions and Answers
**Q1. What is a lambda function?**  
It is an anonymous function defined with the `lambda` keyword.

**Q2. When should lambda functions be used?**  
Use them for short, simple operations. For larger logic, regular `def` functions are clearer.

**Q3. Can a lambda contain multiple expressions?**  
No. A lambda contains only one expression.

## 13. Common Coding Interview Questions

### 1. Reverse a String
```python
def reverse_string(s):
    return s[::-1]
```

**Explanation:** String slicing with step `-1` returns the reversed string.

### 2. Check Palindrome
```python
def is_palindrome(s):
    return s == s[::-1]
```

**Explanation:** A palindrome reads the same forward and backward.

### 3. Find Largest Element in a List
```python
def largest(nums):
    return max(nums)
```

### 4. Count Vowels in a String
```python
def count_vowels(s):
    vowels = "aeiouAEIOU"
    return sum(1 for ch in s if ch in vowels)
```

### 5. Fibonacci Series
```python
def fibonacci(n):
    a, b = 0, 1
    result = []
    for _ in range(n):
        result.append(a)
        a, b = b, a + b
    return result
```

### 6. Factorial
```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

### 7. Check Prime Number
```python
def is_prime(n):
    if n <= 1:
        return False
    for i in range(2, int(n ** 0.5) + 1):
        if n % i == 0:
            return False
    return True
```

### 8. Remove Duplicates from List
```python
def remove_duplicates(lst):
    return list(set(lst))
```

### 9. Find Frequency of Characters
```python
def char_frequency(s):
    freq = {}
    for ch in s:
        freq[ch] = freq.get(ch, 0) + 1
    return freq
```

### 10. Merge Two Dictionaries
```python
def merge_dicts(d1, d2):
    return {**d1, **d2}
```

## Frequently Asked Placement Questions with Sample Answers

### Q1. What is the difference between `==` and `is`?
- `==` checks value equality.
- `is` checks object identity, meaning whether two references point to the same object.

**Example:**
```python
a = [1, 2]
b = [1, 2]
print(a == b)  # True
print(a is b)  # False
```

### Q2. What is slicing in Python?
Slicing is a way to extract part of a sequence using the syntax `sequence[start:stop:step]`.

### Q3. What is the difference between `append()` and `extend()`?
- `append()` adds one object at the end of a list.
- `extend()` adds each element of an iterable to the list.

### Q4. What are shallow copy and deep copy?
- Shallow copy creates a new outer object but nested objects are still referenced.
- Deep copy recursively copies nested objects too.

### Q5. What is duck typing?
Duck typing means Python focuses on an object’s behavior rather than its exact type. If an object behaves like the needed type, it can be used.

### Q6. What is the difference between `pass`, `continue`, and `break`?
- `pass` does nothing and acts as a placeholder.
- `continue` skips the rest of the current loop iteration.
- `break` exits the loop immediately.

### Q7. Why are tuples faster than lists in some cases?
Tuples are immutable, so they generally have lower overhead than lists for fixed collections.

### Q8. What is `enumerate()`?
`enumerate()` returns both index and value while looping.

```python
for index, value in enumerate(["a", "b", "c"]):
    print(index, value)
```

### Q9. What is the difference between `sort()` and `sorted()`?
- `sort()` modifies the list in place.
- `sorted()` returns a new sorted iterable result.

### Q10. What is a Pythonic solution?
A Pythonic solution is one that uses clear, idiomatic Python constructs and emphasizes readability.
